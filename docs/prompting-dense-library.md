## Overview

Dense formulations repository. Systematic prompting approach. Precise directives > verbose instructions.

Repository of **dense formulations** for each ingestion and analysis workflow operation. This systematic prompting approach replaces lengthy verbal instructions with ultra-precise directives increasing clarity and semantic efficiency.

**Core principle**: Maximum semantic density. One precise term > one vague phrase.

**Application**: Ces formulations sont intégrées de manière systématique dans toute la logique du workflow, pas utilisées ponctuellement.

---

## Catégorie 1: EXTRACTION & ANALYSIS

### Extraction Exhaustive

**Formulation dense**: Extract exhaustively. Distill ruthlessly. Zero loss.

**Application**: Phase 0 d'ingestion, analyse de sources brutes

**Logique**: 
- "Extract exhaustively" = directive précise d'extraction complète
- "Distill ruthlessly" = élimination méthodique du superflu
- "Zero loss" = contrainte absolue de préservation

**Résultat**: 
- Capture complète de tous les éléments pertinents
- Élimination du superflu
- Aucune information perdue

**Exemple e-commerce**:
```
Source: YouTube 45 min expert creative strategy
→ "Extract exhaustively. Distill ruthlessly. Zero loss."
→ Output: 20-30 concepts actionnables, formules exactes, exemples complets
```

---

### Analyse Sémantique

**Formulation dense**: Delineate precisely. MECE breakdown. Semantic decomposition.

**Application**: Analyse du sens des concepts, catégorisation

**Logique**:
- "Delineate" = délimiter avec précision (vs "analyser" qui est flou)
- "MECE" = framework structurant (Mutually Exclusive, Collectively Exhaustive)
- "Semantic decomposition" = décomposer par le sens

**Résultat**:
- Délimitation précise de chaque concept
- Décomposition Mutually Exclusive, Collectively Exhaustive
- Séparation des éléments sémantiques

**Exemple e-commerce**:
```
Concept: "Scaling"
→ "Delineate precisely. MECE breakdown."
→ Output: 
  - Vertical scale (augmenter budget)
  - Horizontal scale (dupliquer campagnes)
  - Creative scale (multiplier angles)
(3 concepts distincts, pas 1 concept flou)
```

---

### Analyse Forensic

**Keywords**: `Forensic approach. Root cause. Systematic dissection.`

**Application**: Investigation profonde, compréhension des mécanismes

**Résultat**:
- Approche méthodique et rigoureuse
- Identification de la cause racine
- Décomposition systématique

**Exemple e-commerce**:
```
Problème: "CPA explose"
→ "Forensic approach. Root cause."
→ Output:
  1. Creative fatigue? (analyser CTR/Frequency)
  2. Offre faible? (analyser CVR)
  3. Mauvais timing? (analyser saisonnalité)
  → Root cause identifiée avec données
```

---

### Analyse Comparative

**Keywords**: `Differential diagnosis. Quantify similarities. MECE.`

**Application**: Comparer concepts, détecter duplications, mesurer similarité

**Résultat**:
- Élimination par diagnostic différentiel
- Quantification des similarités (scores)
- Classification exhaustive et exclusive

**Exemple e-commerce**:
```
Nouveau concept vs Index existant
→ "Differential diagnosis. Quantify similarities."
→ Output:
  - "Framework 3i" vs "Itération Créatives": 65% similarité (variante)
  - "Framework 3i" vs "Pattern Spotting": 30% similarité (distinct)
  → Classification: Framework 3i = nouveau concept enrichissant Itération
```

## Catégorie 2: CLASSIFICATION & PLACEMENT

### Placement Intelligent

**Keywords**: `Method flow optimization. Depth constraint (max 4). Hierarchical coherence.`

**Application**: Déterminer où placer un nouveau concept dans la structure

**Résultat**:
- Optimisation selon flux logique méthodologique
- Respect de la contrainte profondeur
- Cohérence hiérarchique parent-enfant

**Exemple e-commerce**:
```
Nouveau concept: "Vision 4D Audiences"
→ "Method flow optimization. Hierarchical coherence."
→ Output:
  - Parent: Meta Ads (domaine régie)
  - Level: 3/4 (tactical)
  - Position: Après basics, avant advanced
  - Placement: 04-acquisition/02-payant/01-regies/01-meta/03-vision-4d-audiences.md
```

---

### Validation Taxonomique

**Keywords**: `Triangular validation (Tax ↔ Sem ↔ Onto). Pinpoint discrepancies.`

**Application**: Vérifier cohérence entre taxonomie, sémantique et ontologie

**Résultat**:
- Validation triangulaire des 3 couches
- Détection précise des incohérences
- Arbitrage par ontologie si conflit

**Exemple e-commerce**:
```
Document: "Internationalisation" dans 03-marketing/
→ "Triangular validation. Pinpoint discrepancies."
→ Output:
  - Taxonomie: Placé dans Marketing (code 03)
  - Sémantique: Levier de croissance (expansion)
  - Ontologie: Domaine = Expansion/Growth
  → INCOHÉRENCE CRITIQUE: Taxonomie ≠ Sémantique/Ontologie
  → Action: Déplacer vers 10-expansion/
```

---

### Détection Cross-Topic

**Keywords**: `Multi-domain scan. Different angles strategy. Functional hierarchy.`

**Application**: Identifier concepts pertinents dans plusieurs domaines

**Résultat**:
- Scan de tous les domaines pertinents
- Stratégie d'angles différents (pas de duplication)
- Identification hiérarchie fonctionnelle (parent-child)

**Exemple e-commerce**:
```
Concept: "Creative Testing"
→ "Multi-domain scan. Different angles strategy."
→ Output:
  - Pertinent dans: Creative (testing créatives) + Acquisition (impact campagnes)
  - Angle Creative: Méthodologies testing (Framework 3i)
  - Angle Acquisition: Impact sur performance (CPA/ROAS)
  → Functional hierarchy: Creative = parent, Acquisition = utilise résultats
  → Pas de duplication, 2 documents avec angles différents
```

## Catégorie 3: QUALITÉ & VALIDATION

### Validation Préventive

**Keywords**: `Systematic verification. Checklist execution. Zero tolerance.`

**Application**: Valider la qualité avant output final

**Résultat**:
- Vérification systématique point par point
- Exécution de checklist complète
- Tolérance zéro pour les erreurs

**Exemple e-commerce**:
```
Après analyse complète
→ "Systematic verification. Zero tolerance."
→ Output (Checklist):
  - [ ] Zero loss: Tous éléments extraits? ✓
  - [ ] Quadruple analysis: Complète? ✓
  - [ ] Depth: ≤4 levels? ✓
  - [ ] Method flow: Respecté? ✓
  - [ ] Metadata: Complètes? ✗ → Source manquante (BLOQUE)
  → Correction requise avant validation
```

---

### Détection Duplication

**Keywords**: `Similarity matrix (>80% threshold). Eliminate ruthlessly.`

**Application**: Détecter concepts redondants, éviter duplication

**Résultat**:
- Matrice de similarité calculée
- Seuil 80% pour duplication
- Élimination sans compromis

**Exemple e-commerce**:
```
Nouveau concept "Media Mix Créatif" vs Index
→ "Similarity matrix (>80% threshold)."
→ Output:
  - vs "Media Mix": 85% similarité
  - vs "Format Créatifs": 40% similarité
  → DUPLICATION détectée (>80%)
  → Action: ENRICHIR "Media Mix" existant, pas créer nouveau
```

---

### Validation Cohérence

**Keywords**: `Bidirectional validation. Cross-reference integrity. Format compliance.`

**Application**: Vérifier cohérence des liens, références, format

**Résultat**:
- Validation bidirectionnelle des liens
- Intégrité des références croisées
- Conformité au format standard

**Exemple e-commerce**:
```
Après création documents
→ "Bidirectional validation. Cross-reference integrity."
→ Output:
  - Framework 3i → Itération: Lien créé ✓
  - Itération → Framework 3i: Lien réciproque ✗ (MANQUANT)
  → Action: Ajouter lien réciproque
```

## Catégorie 4: OUTPUT GENERATION

### Synthèse Rapport

**Keywords**: `Structured synthesis. Actionable only. Rigorous brevity.`

**Application**: Génération du rapport d'analyse final

**Résultat**:
- Synthèse structurée claire
- Uniquement éléments actionnables
- Brièveté rigoureuse (pas de fluff)

**Exemple e-commerce**:
```
Rapport d'ingestion
→ "Structured synthesis. Actionable only. Rigorous brevity."
→ Output:
  - Extraction: Liste exhaustive (pas de résumé)
  - Nouveautés: Analyse quadruple complète
  - Changements: Liste numérotée avec actions précises
  - Validation: Checklist binaire (✓/✗)
  → Pas de "peut-être", "pourrait", "éventuellement"
  → Uniquement: CRÉER, ENRICHIR, DÉPLACER (actions claires)
```

---

### Formulation Proposition

**Keywords**: `MECE organization. Priority ranking (Critical|Important|Complementary).`

**Application**: Proposer changements/restructuration de manière structurée

**Résultat**:
- Organisation MECE des propositions
- Ranking par priorité (3 niveaux)
- Clarté des actions

**Exemple e-commerce**:
```
Restructuration proposée
→ "MECE organization. Priority ranking."
→ Output:
  CRITICAL (P0):
  1. Déplacer "Internationalisation" 03-marketing/ → 10-expansion/
     Impact: -50% découvrabilité | Risque: Faible | Effort: 1h
  
  IMPORTANT (P1):
  2. Ajouter liens Framework 3i ↔ Pattern Spotting
     Impact: +30% contexte | Risque: Nul | Effort: 15min
  
  COMPLEMENTARY (P2):
  3. Améliorer wording "08-iteration.md" → "08-iteration-creatives.md"
     Impact: +10% clarté | Risque: Nul | Effort: 5min
```

---

### Analyse d'Impact

**Keywords**: `Trade-off mapping. Risk-benefit quantification. Execution roadmap.`

**Application**: Évaluer conséquences des changements proposés

**Résultat**:
- Cartographie des compromis (trade-offs)
- Quantification risque/bénéfice
- Roadmap d'exécution par phases

**Exemple e-commerce**:
```
Changement proposé: Déplacer "Internationalisation"
→ "Trade-off mapping. Risk-benefit quantification."
→ Output:
  BENEFITS:
  - Cohérence taxonomique: +90%
  - Découvrabilité: +50%
  - Clarté domaine: +100%
  
  RISKS:
  - Liens cassés: 5 fichiers à mettre à jour
  - Temps migration: 1h
  - Risque erreur: 10%
  
  TRADE-OFF: FAVORABLE (benefit >> risk)
  
  ROADMAP:
  Phase 1: Créer 10-expansion/01-internationalisation/ (5min)
  Phase 2: Déplacer fichier + update liens (30min)
  Phase 3: Valider aucun lien cassé (10min)
  Phase 4: Update index concepts (5min)
```

## Catégorie 5: MÉTHODOLOGIES SPÉCIFIQUES

### First Principles Thinking

**Keywords**: `Deconstruct to axioms. Rebuild from foundation. Question assumptions.`

**Application**: Comprendre la vraie nature d'un concept

**Résultat**:
- Déconstruction jusqu'aux axiomes
- Reconstruction depuis la base
- Questionnement des assumptions

**Exemple e-commerce**:
```
Concept: "Creative is New Targeting"
→ "Deconstruct to axioms. Question assumptions."
→ Output:
  AXIOM 1: Ciblage = Montrer pub à bonne audience
  AXIOM 2: Pré-iOS14: Ciblage technique (data Facebook)
  AXIOM 3: Post-iOS14: Perte data → ciblage technique limité
  AXIOM 4: Creative peut attirer spécifiquement un avatar
  → CONCLUSION: Creative effectue maintenant le ciblage (hook spécifique)
  → Principe reconstructed from foundation
```

---

### Root Cause Analysis

**Keywords**: `Five Whys. Eliminate symptoms. Isolate true lever.`

**Application**: Identifier la cause racine d'un problème/succès

**Résultat**:
- Méthode des 5 Pourquoi
- Élimination des symptômes
- Isolation du vrai levier

**Exemple e-commerce**:
```
Problème: "CPA trop élevé"
→ "Five Whys. Isolate true lever."
→ Output:
  1. Pourquoi CPA élevé? → CTR faible
  2. Pourquoi CTR faible? → Hook ne capte pas attention
  3. Pourquoi hook faible? → Pas adapté à l'avatar
  4. Pourquoi pas adapté? → Deep research insuffisant
  5. Pourquoi insuffisant? → Pas fait de Phase 0
  → ROOT CAUSE: Absence Deep Research
  → TRUE LEVER: Implémenter Phase 0 avant créatives
```

---

### MECE Breakdown

**Keywords**: `Mutually Exclusive, Collectively Exhaustive. No overlap, no gap.`

**Application**: Décomposer un domaine/concept de manière structurée

**Résultat**:
- Catégories mutuellement exclusives
- Exhaustivité collective (rien n'est oublié)
- Pas de chevauchement, pas de lacune

**Exemple e-commerce**:
```
Décomposer "Scaling Strategies"
→ "MECE breakdown. No overlap, no gap."
→ Output:
  SCALING (parent):
  ├─ Vertical Scale (augmenter budget campagne existante)
  ├─ Horizontal Scale (dupliquer campagnes/audiences)
  └─ Creative Scale (multiplier angles/formats)
  
  Validation MECE:
  - Mutually Exclusive? ✓ (aucun overlap)
  - Collectively Exhaustive? ✓ (toutes stratégies couvertes)
```

---

### Socratic Method

**Keywords**: `Question systematically. Reveal unstated assumptions. Deepen understanding.`

**Application**: Approfondir un concept par questionnement

**Résultat**:
- Questionnement systématique
- Révélation des assumptions non-dites
- Approfondissement de la compréhension

**Exemple e-commerce**:
```
Concept: "Angle Marketing"
→ "Question systematically. Reveal unstated assumptions."
→ Output:
  Q1: Qu'est-ce qu'un angle? → Promesse spécifique à un avatar
  Q2: Différence avec mécanisme? → Angle = quoi, Mécanisme = comment
  Q3: Combien tester? → 3-5 par avatar
  Q4: Quand changer? → Si CPA >120% break-even après 7 jours
  Q5: Dependency? → Nécessite Deep Research (connaître désir profond)
  → Understanding deepened with 5 critical questions
```

## Catégorie 7: E-COMMERCE SPÉCIFIQUE

### Profitabilité vs Scalabilité vs Durabilité

**Keywords**: `Profitable (break-even). Scalable (growth efficient). Sustainable (resilient).`

**Application**: Classer concepts selon phase business

**Résultat**:
- Phase 1: Profitabilité (atteindre break-even)
- Phase 2: Scalabilité (croissance efficiente)
- Phase 3: Durabilité (système résilient)

**Exemple e-commerce**:
```
Classifier concepts selon phase
→ Output:
  
  PROFITABLE (Phase 1):
  - Validation product-market fit
  - Break-even CAC calculation
  - TAM estimation
  → Placement: 02-product/ (fondamentaux)
  
  SCALABLE (Phase 2):
  - Vertical/Horizontal scale
  - Framework 3i
  - Structure campagnes
  → Placement: 04-acquisition/, 05-creatives/
  
  SUSTAINABLE (Phase 3):
  - LTV optimization
  - Backend infrastructure
  - Team recruitment
  → Placement: 08-retention/, 09-infrastructure/
```

## Guide d'Utilisation

### Comment Choisir les Magic Keywords?

**Étape 1**: Identifier l'opération
- Extraction? → Catégorie 1
- Classification? → Catégorie 2
- Validation? → Catégorie 3
- Output? → Catégorie 4

**Étape 2**: Sélectionner combinaison appropriée
- Consulter cette bibliothèque
- Choisir 2-3 keywords max par opération

**Étape 3**: Intégrer dans prompt/template
- Placer en début d'instruction
- Laisser keywords guider l'action

**Exemple**:
```
❌ AVANT (verbeux):
"Veuillez extraire tous les concepts de manière exhaustive sans perdre aucune information et en préservant tous les détails"

✓ APRÈS (magic keywords):
"Extract exhaustively. Zero loss."
```

---

### Règles d'Or

1. **Maximum 3-4 keywords par instruction** (au-delà = dilution)
2. **Keywords en début d'instruction** (position privilégiée)
3. **Pas de redondance** (1 keyword = 1 concept)
4. **Référencer cette library** au lieu de réinventer
5. **Tester et mesurer** l'impact sur qualité outputs
